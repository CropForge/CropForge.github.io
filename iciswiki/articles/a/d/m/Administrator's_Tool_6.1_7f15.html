Main components

1. Configuration

  • interface for viewing ODBC info, viewing/editing INI settings

2. User Maintenance (STUDY-level)

  • interface for managing user access to studies (A user must be a member of a project to view studies in that project. A user may belong to several projects. A study belongs to only one project.)
  • assign a user to one or more projects
  • assign a study to a project

Access restrictions are application-level only (not in database)

Schema

1. New table PROJUSER in DMS to contain user-project matching
2. The STUDY table (ICIS schema v5.5) has a PMKEY column (but currently not being used), to contain the project management ID.
3. Add entries to UDFLDS table with FTABLE = 'PROJUSER' and FTYPE = 'PMKEY' to define matching of a project management ID to a project name.
4. Contents of original STUDY table are copied to a temporary table M_STUDY. STUDY table is emptied then populated with studies that a user has access rights to.

3. Update Central

  • interface for uploading data from a local ICIS database to a central ICIS database
  • interface for deleting studies from the central ICIS database
