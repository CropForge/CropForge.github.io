Overview

This use case allows direct inclusion of incoming sample to the collection without passing through the initial seed increase. The information needed are the temporary ID, the result viability test, and the amount of original seed packets received.

The user is provided with a form which she can fill in to specify the amount of seeds to be stored in the active and in the base collection. He must also supply the inventory for the paper packets, the bulk and the remnants.

Use Case Definition

Use Case Name 3.0.0 Direct Assignment of Accession Number to Incoming Sample
Use Case Definition The use case allows registration of newly acquired samples to be registered directly as IRGC accession.
User Contacts n/a
Actors Genbank Manager
Location Genebank
Priority 2
Typical Course of Events
Actor Action System Response
Step 1: The actor supplies the temporary identification number of germplasms that needs to be processed and registered directly as IRGC accession. Step 2: System retrieves the seed info of the temporary identification numbers specified by the actor.
Step 3. Actor fills in the provided user entry form (i.e. viability, inventory data) Step 4. System stores the information in the database.
Step 5. Actor requests for a prooflist. Step 6. System outputs the report.
Step 7. Actor executes the process button. Step 8. System automatically generates the accession number.
Step 7. Actor concludes the use case.
Assumption/s The incoming seed information is complete and accurate
Pre-condition/s pre-determined samples or list of temporary accesison number for direct assignment of accession number.
Post-condition/s Assigned accession number to incoming sample
Primary Pathway/s
Alternative Pathway/s

1)Plant the materials for Initial Seed Increase

2)Direct assignment of accession number

Exception Pathway/s n/a
