Downloading and Installing ICIS 5.5

1) Connecting to FTP site Run any FTP program and connect to the ICIS FTP site by specifying the URL (FTP address) as 198.93.224.42 (ftp.cgiar.org). If asked for a username and password, enter anonymous as username and your complete e-mail address as the password.

2) Accessing ICIS Ver5.5_Directory Change the current logged-on directory to /ICIS/Ver5.5_Release. This directory contains self extracting compressed files for the software and the different crop implementations. ICISPROG.EXE : This file contains the ICIS software and technical development manual.
TUTORIAL.EXE : This file has the Tutorial data. IRIS-GMS.EXE and IRIS-DMS.EXE : These files are crop databases for rice. IWIS3-GMS.EXE and IWIS3-DMS.EXE : These files are crop databases for wheat. IBIS.exe : Crop database for barley. IPotIS.exe : Crop Database for potato. Etc..

3) Backup existing files and Download required new files The setup process described below may overwrite some files depending on paths which are specified. If you have existing local databases which should be retained, back them up by copying them to a backup subdirectory such as C:\ICIS5\backup. You can choose another drive and root if you prefer but \ICIS5\is recommended. Now you may uninstall the old version of ICIS if it was set up by the ICIS installer by using the Add/Remove Programs function on the Windows Control Panel.
Next, select the ICISPROG.exe file and the files for the crop database(s) you require. Then download them to any temporary directory on the target computer, such as C:\ICIS5\setup

4) Installing central databases After downloading is complete, run (double click) the database EXE program(s) to extract the contents of the required crop database(s). These programs ask the user to specify the path to the directory where ICIS is to be installed. C:\ICIS5 is proposed, but another drive may be specified. This will be the directory where ICIS will be installed.

5) Installing Documentation The ICIS Ftp site also contains self extracting zip files for ICIS Documentation. To install this documentation in the ICIS tree, download the file ICISDOCS.EXE and run it in the same way as in step 3.

6) Installing ICIS software Run the ICISPROG.EXE program (double click). This will ask for the ICIS directory as in setup 3 and then unpack the ICIS software. Next, go to the setup folder and run the ICIS Setup program Setup.EXE (See Installation notes). Part of the setup process is to specify local installations. If you want to install a new local installation select “Create a new installation” and specify any new username, password and file name. If you are re-installing and existing installation for which all data has been uploaded into the new central database select “Update an existing installation” and then pick the installation from the list and give the correct username, password. If you have local data for any installation which must be re-installed follow step 7, otherwise go to step 10.

7) Reinstalling existing local installations with local data Run setup.exe for each local installation for which you have existing data backed up in the \backup subdirectory (step 2).
If you are re-installing and existing installation which has already been registered with the central database, select “Update an existing installation” and then pick the installation from the list and give the correct username, password and the same file name (not file path) as the backed-up copy.
If the installation has not been registered then select “Create a new installation” but give the username, password and file name (not file path) of your existing database (the username and password can be found in the USERS table of your backed-up local database).

8) To restore data in existing local databases copy the backed-up files from their backup location (\ICIS5\backup) to the appropriate ICIS5\Database\<crop>\Local subdirectory which will have been created during step 5). They should overwrite the new (empty) local databases created during step 7 (rename the old files to match the new ones if necessary).

9) Converting your old local databases to version 5.0 Finally you must make sure that these old database files have the V5.0 structure. For conversion from v3 refer to the Technical Documentation of 5.4 ).
For conversion from 4.1 to 5.0 set the field LISTDATA.ENTRYCD to Text 47, check that UDFLDS table has a field called SCALEID of type long integer and that the BIBREFS table has field PUBDATE (Long Integer) in place of PUBYEAR.

10) When installation is complete, all downloaded files plus the ICIS5\setup directory and its contents can be deleted.

11) ICIS directory structure The ICIS directory tree, which will have been set up, contains the template databases and central and local databases specified during installation in branches of the subdirectory \DATABASE (Figure 5.1.1). The setup program will have established ODBC data source names (5.2.3b) for local and central databases and ICIS.INI files for each installation specified (5.3.1).

12) Running ICIS A link to the launcher will have been placed on the desk. This can be executed immediately (5.3.2). The launcher should be referring to the latest installation. You can use the Change INI button to change reference to another installation if you have more than one.
