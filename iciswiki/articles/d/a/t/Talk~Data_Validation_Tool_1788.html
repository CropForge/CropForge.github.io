Validation checks for all ICIS implementations

Check 1

The GDate of a GID must not predate GDate of any of its progenitors. (Beware missing links in the chain of dates! If the test checks only the GDATE of the progenitors GPID1, GPID2 and MGID, then it will not detect the following error: non-zero GPID1, GPID2 or MGID has GDATE=0 but their GPID1, GPID2 or MGID are younger than the target GID. Therefore, if a GPID1, GPID2 or MGID has GDate=0, iterate to check their GPID1, GPID2 and MGID)

Check 2

If a GID inherits a name from its source (GPID2) then that name record must also inherit the NDATE and NLOCN

Check 3

A GID can’t have GPID2>0 and GPID1=0

Validation checks for IRIS

Check 4

Exactly one name (neither more nor less) of a GID must have NSTAT=1 (preferred English name)

Check 5

Names eligible to be the preferred name are: CRSNM, RELNM, DRVNM, CVNAM, ELITE

Check 6

No more than one name of a GID can have a "preferred ID" status (NSTAT =8)

Check 7

The following name types, if present, must be preferred ID: ACCNO, GACC, ITEST, CIATGB

Check 8

The following names types are eligible to be preferred ID: DRVNM, COLNO

Check 9

The preferred ID must be unique for the given name type – that is, two accessions must not share the same name of the same type if one or both is a preferred ID.

Check 10

For all methn≠62, GLOCN of a GID = GLOCN of its GPID2

Check 11

  • Only certain combinations of name type and germplasm creation method are acceptable, namely
    • Method type GEN: valid name types are: CRSNM, UNCRS, UNRES
    • Method type DER: valid name types are: RELNM, DRVNM, CVNAM, CVABR, NTEST, LNAME, ADVNM, ACVNM, AABBR, OLDMUT1, OLDMUT2, ELITE, UNRES
    • Method type MAN: valid name types are: ACCNO, RELNM, CVNAM, CVABR, COLNO, FACCN, ITEST, NTEST, LNAME, TACC, ADVNM, ACVNM, ELITE, GACC, DACCN, LCNAM, CIATGB

Check 12

  • Name type ACCNO, CIATGB and ITEST must
    • have METHN=62
    • GLOCN = location of holding organization
    • GPID2≠0 = GID of sample held by donor, with GLOCN of GPID2=location of donor

Check 13

  • Some name types must not occur more than once for a single GID. These are ACCNO, CRSNM, UNCRS, COLNO, ITEST, GACC, CIATGB, RELNM, DRVNM, CVNAM

Check 14

  • RELNM cannot occur more then once for a GID-GLOCN-COUNTRY combination. Two GIDs can share the same RELNM only if their GLOCNs are in different countries

Check 15

  • Some name types cannot occur together, e.g. (Cross-names with line-names), (RELNM with COLNO)

Check 16

  • If a GID has methn=62,
    • it should inherit all names from its source, together with their NLOCN
    • Inherited names of type ACCNO, CIATGB, ITEST should change to type DACCN
    • Inherited names of type DACCN should change to type FACCN
    • Inherited names of type RELNM should change to type CVNAM
    • Other names should inherit the name type unchanged
    • The status of preferred name should be inherited
    • The status of preferred ID must not be inherited.

Check 17

  • Conversely, if GID A has a name of type ACCNO, and GID B has the same name of type DACCN, then
    • GID B must be the source of GID A
    • GID A must have METHN=62

Check 18

  • A GID with MTYPE=MAN or MTYPE=DER cannot be the GPID1 of any other GID unless it has GPID1=GPID2=0

Check 19

  • All names of type GACC must have
    • a methn with MTYPE=MAN
    • MGID≠0
    • GPID1=MGID if its MGID has GPID1=0
    • GPID1=GPID1 of its MGID not 0

Check 20

  • If a GPID1 has a name of type COLNO, then it represents a specific known sample collected from the field, and
    • It must have a GLOCN of type collecting location (even if the only thing known about it is the country)
    • It must have a METHN=69 = collection
    • All GIDs that have it as their GPID1 must also have the same name of type COLNO
    • The COLNO name of the GPID1 must be its preferred ID
    • The COLNO of all GIDs sharing this common GPID1 must not be preferred ID

Check 21

  • If a GPID1 has a preferred name of type CVNAM with no COLNO, then it represents a landrace sample collected from the field but with unknown collector's ID. In this case,
    • No other GID can share the same GPID1
    • It must have a METHN=69 = collection
    • The GPID1 must have its GPID1=GPID2=0

Check 22

  • If a GPID1 has a name of type RELNM, then it represents an official release of a variety in a country with unknown pedigree (if it had known pedigree, it couldn't itself be a GPID1. Then
    • All GIDs that have it as their GPID1 must have that same value of the RELNM, but with name type CVNAM
    • The GPID1 must have its GPID1=GPID2=0

Desirable data validation not yet possible

For names that comprise an alphabetic code followed by a number (e.g. IRGC 1234, PI 154452, IRTP 12345), we should assemble an additional table of

CodeOrganizationName type
IRGCIRRI GRCACCNO
The code used as the start of the nameThe location of the institute that first assigns names of this format The name type assigned by the naming institute, e.g. ACCNO, CRSNM, ITEST, CVNAM, RELNM

Such a table would allow us to systematically validate and correct the NLOCN of such names.

Validation checks for IRGC accessions

  • A NAME type GACC beginning “IRGC” must have
    • the format “IRGC nnnnn:yyyyss”,
    • yyyy=year(NDATE)=year(GDATE) of its GID
    • “IRGC nnnnn”= the name (type ACCNO) of its MGID
    • ss=DS or WS or SS or RS
