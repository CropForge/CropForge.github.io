Introduction

This page descibes a proposed extension to the DMS schema version 6.0 specifically for high throughput (HTP) genptyping data. Due to the large size of these HTP datasets it may not be possible to store and retrieve efficiently these datasets in the standard DMS data model. The following extended data model has been proposed to cope with these datasets in a simple and efficient way with direct links to GMS, GEMS and IMS. If this extension provides better support for genotyping data then the following text will be move to the main page for DMS schema version 6.0.

Extended Data Model

The extended data model provides for the additional concept for scoring genotyping data. This concept is the genoptype data point, which has the following attributes:

  • Identifier - Unique identifier of the data point score.
  • Study - The study in which the data point was measured.
  • Sample - Optional identifier of the DNA sample, if not defined it was assumed that the sample was not recorded independantly from the germplasm entry.
  • Germplasm - The germplasm entry measured in this data point.
  • Genome Feature Detector - The genome feature detector (marker) used in this data point. This attribute is redudent since it is possible to determine the Genome Feature Detector from the Detected Variant, however it is included here to speed up queries by Genome Feature Detector.
  • Detected Variant - The Detected Variant measured or found by the Genome Feature Detector for this data point
  • Incidence - The indicates the frequency of that the Detected Variant in the sample relative to other alleles. In the case of dominant markers or co-dominant markers using a genotyping platform where it is not possible to determine the relative frequency of a Detected Variant for a given Genome Feature Detector, this incidence would be '1' indicating the presence of only this allele or '0' indicating the absence of the allele. Where the genotyping platform in use can determine the relative frequency of the allele present in the sample, the incidence will be the allele frequency with '1' indicating the presence of only this allele in homozygote form
    '0.5' indicating the presence of only this allele in hetrozygote form or '0' indicating the absence of
    the allele. In the case of bulked DNA samples taken from segregating or open polenated variety it may be possible to have a incidence (relative frequency) ranging from 0 to 1, inclusive. Any value outside of the this range is considered as 'missing data', however null is normally used to denote missing data point of non-specified cause.

Table Definitions

DMS_GENOTYPE_SCORES

The DMS_GENOTYPE_SCORES table contains one entry for a single genotyping score of a specific DetectedVariant:' ("allele") of a GenomicFeatureDetector:("marker")

Please look at Proposed GEMS 6.0 schema for more details on these concepts

Table 1. DMS_GENOTYPE_SCORES table

Fieldname Description Definition (MySQL)
DATA_POINT_ID Unique identifier for the data point
 INTEGER[11], PRIMARY KEY, AUTO_INCREMENT
STUDYID Unique identifier of the Study or folder of study INTEGER[11], FOREIGN KEY STUDY(STUDYID)
SAMPLEID Reference to sample from which is score was recorded. INTEGER[11], FOREIGN KEY GERMPLSM(GID)
GID Reference to germplasm in which score was recorded INTEGER[11], FOREIGN KEY GERMPLSM(GID)
GFDETECTORID Reference to genomic feature detector used INTEGER[11], FOREIGN KEY GEMS_GENOMIC_FEATURE_DETECTOR(GFDETECTORID)
GVARIANTID Reference to detected variant scored INTEGER[11], FOREIGN KEY GEMS_DETECTED_VARIANT(GVARIANTID)
INCIDENCE DOUBLE[11] UNSIGNED The incidence of the the detected varient in the specified germplasm for the given study

GEMS Version 5.4 support

The above table can also be implemented at first using the GEMS 5.4 schema using the following modifications

Fieldname Description Definition (MySQL)
GFDETECTORID INTEGER[], FOREIGN KEY GEMS_MD(MDID) Reference to polymorphism detector
GVARIANTID INTEGER[], FOREIGN KEY GEMS_MV(MVID) Reference to molecular variant
