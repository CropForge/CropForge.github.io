Overview

Step 3: Request for distribution


Use Case Definition

Use Case Name 1.1.1 Request Seed for Distribution
Use Case Definition The use case allows the actor to request seed for distribution. This is accessed using standalone application and external request shall be accommodated using different [web] interface.
User Contacts n/a
Actors Genebank Seed Export Secretary
Location Genebank
Priority 1
Typical Course of Events
Actor Action System Response
Step 1: The use case is initiated when the actor specify the request type (e.g. within IRRI/ outside IRRI) and the accession number(s) of the germplasm requested. Step 2: The system retrieves information and checks the availbility of the accession(s) for distribution.
Step 3: The actor must supply the requestor details and the request purpose. Step 4: The system creates the seed list using listname as the request number.
Step 5: The use case is concluded when the actor receives the final confirmation from the system that the request is being processed and is waiting for approval of the Genebank manager.
Assumption/s Determined accession number
Pre-condition/s
Post-condition/s Request information is stored in the database
Primary Pathway/s Request of seeds for distribution
Alternative Pathway/s n/a
Exception Pathway/s n/a
