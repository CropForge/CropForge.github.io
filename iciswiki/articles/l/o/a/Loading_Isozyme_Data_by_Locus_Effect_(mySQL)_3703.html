DATA PREPARATION

The following procedure assumes that the steps in loading data by allele effect was completed.

Create ISOSER_LOCUS table

ISOSER_LOCUS table will contain the serial representation for locus data. 

CREATE TABLE isoser_locus (orderno mediumint(11) not null auto_increment unique,ounitid int,sampleno double precision, gid 
int, name varchar(255), origin int, source varchar(255),locus varchar(50),locid int,
gtpe  varchar(10),snvalue double precision, snlevel int, locvalue double
precision,loclevel int, KEY samplex(sampleno),KEY locidex (locid));

DEFINE THE SAMPLE BY LOCUS EFFECT

The data in this effect will be the GTPE value of each LOCUS in each sample

Add GTPE in the VARIATE table

Since,allele effect was previously added to the variate table, GTPE will now have -2 as variatid. 

INSERT INTO variate ( variatid, vname, studyid, vtype, traitid, scaleid, tmethid,dtype )
       VALUES (-2, 'GTYPE', -1, 'DV', 505, 513, 519, 'N');

Add Locus factor in the FACTOR table

labelid for the locus data is -9 since the allele effect added 8 labelids in the FACTOR table. 

INSERT INTO factor ( labelid, factorid, fname, studyid, traitid, scaleid, tmethid, ltype )
            VALUES (-9, -9, 'LOCUS', -1, 506, 523, 524, 'N');

Define the Sample by locus effect in the EFFECT table

  • STUDY-- 
INSERT INTO effect ( represno, factorid, effectid ) VALUES (-3, -1, -3);
  • SAMPLENO-- 
INSERT INTO effect ( represno, factorid, effectid ) VALUES (-3, -2, -3);
  • LOCUS-- 
INSERT INTO effect ( represno, factorid, effectid ) VALUES (-3, -9, -3);

Locus effect is another representation of the same study (ISOZYME04). The locus effect can be represented by all combinations of levels of factors -1(study), -2(Sampleno) and -9 (locus).

Set up the LOCUS factor

  • Add levelno Column to ISO_LOCUS table. 
ALTER TABLE iso_locus ADD COLUMN levelno int4;
  • Update the levelno in the iso_locus (contains the labels for LOCUS factor) table. Get the minimum of levelno in LEVEL_C and LEVEL_N and use that as an offset value. To update the levelno in each record, subtract locusid to the offset value. The new levelno will now be used for updating the LEVEL_N/LEVEL_C tables 
UPDATE iso_locus
SET levelno = (SELECT Min(mina) AS minlevel 
             FROM (SELECT min(levelno) AS mina FROM level_n 
             UNION 
             SELECT min(levelno) AS mina FROM level_c ) AS c) - locusid;
  • Append locusid factor to LEVEL_N table 
INSERT INTO level_n( labelid, factorid, levelno, lvalue ) 
SELECT -9 AS Expr1, -9 AS Expr2, iso_locus.levelno, iso_locus.locusid FROM iso_locus ;
  • Append LOCUS factor to LEVEL_C table 
INSERT INTO level_c( labelid, factorid, levelno, lvalue ) 
SELECT -10 AS Expr1, -9 AS Expr2, iso_locus.levelno, iso_locus.lname FROM iso_locus;

SET UP THE OBSERVATION UNITS FOR THE SAMPLE LOCUS

Update the snvalue,snlevel,locvalue and loclevel in the ISOSER_LOCUS table

  • Update locid (locus value) in the isoser_locus table. gems_locname table contains the locus name and locus id value. 
UPDATE isoser_locus, gems_locname 
SET locid = gems_locname.locusid 
WHERE isoser_locus.locus = gems_locname.lnval
  • Update loclevel field in the isoser_locus table. In step 2.4, locus factor was added to the LEVEL_N table. To update the loclevel column in the ISOSER_LOCUS table, get the levelno of each sampleno in the level_n table with labelid = -9 (locus factor label) 
UPDATE isoser_locus,level_n
SET loclevel = level_n.levelno 
WHERE (isoser_locus.locid = level_n.lvalue)  AND  (level_n.labelid=-9) ;
  • Update snlevel in isoser_locus by getting the levelno of each sampleno in the level_n table with labelid = -2 (sampleno factor label) 
UPDATE isoser_locus,level_n
SET snlevel = level_n.levelno
WHERE ((isoser_locus.sampleno = level_n.lvalue ) AND (level_n.labelid=-2));
  • Update locvalue field in the isoser_locus table. Values in this column should be equal to isoser_locus.locid value 
UPDATE isoser_locus,level_n
SET locvalue = level_n.lvalue 
WHERE (level_n.labelid=-9) AND (isoser_locus.locid = level_n.lvalue);
  • Update snvalue field in the isoser_locus table. Values in this column should be equal to isoser_locus.sampleno value 
UPDATE isoser_locus ,level_n 
SET snvalue = level_n.lvalue
WHERE (level_n.labelid=-2) AND (isoser_locus.sampleno = level_n.lvalue) ;

Update OUNITID field in ISOSER_LOCUS table

  • Get the minimum of OUNITID from the OINDEX table and update OUNITID field in the isoser_allele table. Updating the 681274 records in the table will take up approx. 80 seconds. 
UPDATE isoser_locus
SET ounitid = (SELECT min(c.ounitid) as mino from oindex as c)-isoser_locus.orderno;

Append STUDY,SAMPLENO and LOCUS observation units to the OINDEX tables.

This will append 2043822 records in the the OINDEX table in approx. 1.5 mins.

  • Append observation units for the study factor 
INSERT INTO oindex ( ounitid, factorid, levelno, represno ) 
SELECT isoser_locus.ounitid, -1 AS Expr3, -1 AS Expr2, -3 AS Expr1 
FROM isoser_locus;
  • Append observation units for the SAMPLENO factor 
INSERT INTO oindex ( ounitid, factorid, levelno, represno ) 
SELECT isoser_locus.ounitid, -2 AS Expr3, isoser_locus.snlevel, -3 AS Expr1 
FROM isoser_locus;
  • Append observation units for the LOCUS factor 
INSERT INTO oindex ( ounitid, factorid, levelno, represno ) 
SELECT isoser_locus.ounitid, -9 AS Expr2, isoser_locus.loclevel, -3 AS Expr1 
FROM isoser_locus;

ADD GTPE IN THE DATA_C TABLE

INSERT INTO data_c (ounitid,dvalue,variatid)
SELECT isoser_locus.ounitid,isoser_locus.gtpe, -2
FROM isoser_locus
